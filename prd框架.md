# 官网功能与能力层级结构文档


## 1. 路由架构 (Routing Structure)

目前系统共包含 5 个核心路由节点，构建了“总-分”式的访问路径：

*   **`/` (Home 首页)**
    *   系统的统一入口，主要用于整体愿景展示与核心模块的分流。
*   **`/list` (方案与案例列表页)**
    *   全量能力落地的承接页。展示所有的“解决方案”与“落地案例”。
    *   支持高度定制化的多维筛选引擎。
*   **`/detail/:id` (通用案例详情页)**
    *   用于承接具体落地案例（Case）的详细说明，包含背景、交付形式、技术标签等明细。
*   **`/solution/:id` (解决方案详情页)**
    *   用于承接核心大方案（Solution）的深度解析页，可能会展示更宏观的架构、痛点分析以及子能力矩阵。
*   **`/capabilities` (技术能力图谱页)**
    *   以纯技术视角（剥离具体客户场景）展示底层的 AI 核心算法库、模型能力与原子技术能力集合。

---

## 2. 筛选引擎能力 (Filtering Capabilities)

列表页（`/list`）作为核心内容枢纽，具备以下动态多维过滤与检索能力：

### 2.1 数据分型筛选（Type Filter）
对数据类型的物理区隔：
*   **全部**：无差别检索
*   **解决方案**：仅筛选具有宏观指导意义与全链路属性的核心 `Solution`
*   **落地案例**：仅筛选具备具体场景属性与客户落地的 `Case`（包括原本的底层能力也作为场景化案例在此呈现）

### 2.2 多维业务筛选（Dimension Filter）
基于数据库字典动态解析的业务层级过滤：
*   **全部行业 (Industry)**
    *   根据数据池中的 `targetIndustry` 字段动态聚合。
    *   例如：*通用/跨行业*、*医疗健康/专科医院*、*产业园区/科创园区运营* 等。
*   **全部业务标签 (Business/Tech Labels)**
    *   根据数据池中的 `techLabels` 字段动态聚合、去重。
    *   涵盖细分的业务场景词或原子能力词。例如：*机器翻译*、*短视频制作*、*智能分诊*、*企业赋能* 等。

### 2.3 文本搜索与排序规则 (Search & Sort)
*   **模糊关键词搜索**：在结果集内对“项目名称（nameZh）”与“一句话简介（shortDescription）”进行字符串匹配检索。
*   **排序机制**：
    *   按**最新更新 (Latest)**：基于 `updateDate` 时间戳降序。
    *   按**相关度 (Relevance)**：升序/算法匹配。

## 3. 能力层级结构 (Capability Hierarchy)

系统在数据定义与业务逻辑上，严格遵循以下 **三级能力下钻结构**：

### 📈 L1: 核心解决方案 (Solutions)
*   **定位**：面向大行业或大领域的宏观“解题思路”。
*   **特征**：包含完整的客户痛点分析、架构全景图。
*   **当前实例**：
    1.  G11N AI 多模态内容出海与本地化解决方案
    2.  智慧医疗与专科临床 AI 解决方案
    3.  智慧园区 AI 底座与企业赋能解决方案

### 📊 L2: 落地案例 (Cases)
*   **定位**：L1 方案在具体客户、具体场景中的物理投射与实施。
*   **特征**：具备明确的交付形式（如混合模式、SaaS）、项目背景、保密级别等实施属性。
*   **当前实例**：
    *   *AI翻译质量审核*（归属 L1 的 G11N 方案）
    *   *智能分诊及术后随诊*（归属 L1 的 医疗方案）
    *   *科学之门AI中台*（归属 L1 的 园区方案）

### 🧩 L3: 原子技术与业务标签 (Tech & Business Labels)
*   **定位**：支撑 L1 和 L2 的最底层细分能力。
*   **特征**：高度复用、高度碎片化。作为过滤标签附着于上层实例。
*   **当前实例**：
    *   底层技术（Underlying Tech）：*LLM*、*知识图谱 (KG)*、*容器化 (K8s)*、*RAG*。
    *   业务标签（Labels）：*多租户*、*质量评估*、*临床决策支持*。

## 4. 用户访问路径 (User Journey & Navigation Paths)

---

### 4.1 核心用户旅程地图

#### 路径 A：从"搜索意图"出发（直接检索型）
```
首页 (/)
  │
  ├─[在 Hero 搜索框输入关键词] ──────────────→ 列表页 (/list?keyword=xxx)
  │                                                  │
  │                                         [筛选行业/标签/类型]
  │                                                  │
  │                                    ┌─────────────┴─────────────┐
  │                                    ↓                           ↓
  │                           案例详情页                    解决方案详情页
  │                       (/detail/:id)               (/solution/:id)
  │
  └─[点击热门搜索词] ──────────────────────→ 列表页 (/list?keyword=热门词)
```

#### 路径 B：从"方案浏览"出发（探索型）
```
首页 (/)
  │
  ├─[向下滚动浏览方案卡片区]
  │       │
  │  [点击"查看方案详情"按钮] ──────────────→ 解决方案详情页 (/solution/:id)
  │       │                                          │
  │  [点击方案下方案例卡片] ──────────────────→ 列表页（带方案关键词筛选）
  │
  └─[点击 Footer 中"查阅落地案例"] ────────→ 列表页 (/list)
```

#### 路径 C：从"技术能力"出发（技术探索型）
```
首页 (/)  或  导航栏
  │
  └─[点击"前往能力全景库"或导航"能力图谱"] ──→ 技术能力图谱页 (/capabilities)
                                                      │
                                              [浏览 L3 原子能力标签]
                                                      │
                                    [点击标签跳转到列表页带该标签筛选]
                                                      │
                                              列表页 (/list?label=xxx)
```

#### 路径 D：从"导航栏直达"（快速跳转型）
```
导航栏 (Navbar) — 始终悬浮可见
  ├─ Logo / 首页链接 ────────────────────────→ 首页 (/)
  ├─ 解决方案 ────────────────────────────────→ 首页 /#solutions 锚点区
  ├─ 能力图谱 ────────────────────────────────→ /capabilities
  └─ 全部案例 ────────────────────────────────→ /list
```

## 5. 页面详细说明 (Page Specifications)

### 5.1 首页 `/` (Home)

**页面定位**：系统统一入口，承担"品牌形象展示 + 快速分流"双重职能。

| 区块名称 | 类型 | 核心内容 | 主要交互 |
|----------|------|----------|----------|
| **Hero Banner** | 全屏锁屏区 | 平台名称 "Global AI Capability Hub"、博彦科技副标题、超大搜索框 | 输入关键词后跳转 `/list`；点击热门搜索词直达列表 |
| **热门搜索词条** | 快捷入口 | 预置高频词：术语管控、知识图谱、多模态出海、数字孪生 | 点击后携带关键词跳转 `/list` |
| **方案展示区（Screen 2–4）** | 全屏滚动卡片 | 每个 L1 方案占一屏，包含方案名、英文名、服务对象、核心技术标签、配图 | 点击"查看方案详情"→ `/solution/:id`；方案底部案例走马灯点击 → `/list?keyword=xxx` |
| **页面 Dot 导航** | 右侧锚点导航 | 小圆点指示当前所在方案屏，共 1+3=4 个点 | 点击圆点定位到对应全屏区域 |
| **底部 CTA 区** | 召唤行动区 | 文案引导 + 两个跳转按钮 | "前往能力全景库"→ `/capabilities`；"查阅落地案例"→ `/list` |

**页面特殊规则**：
- 页面为全屏竖向滚动吸附（scroll-snap），每次滚动定格到一屏
- Hash 路由支持 `#solutions` 锚点直达第一个方案屏

---

### 5.2 列表页 `/list` (List)

**页面定位**：全量内容的核心枢纽，支持多维过滤与检索，是售前与交付人员使用频率最高的页面。

| 区块名称 | 类型 | 核心内容 | 主要交互 |
|----------|------|----------|----------|
| **顶部筛选栏** | 固定筛选工具栏 | 数据分型（全部/方案/案例）、行业下拉、业务标签下拉、关键词搜索框、排序切换 | 选择任意过滤器即时更新结果集 |
| **内容卡片区** | 瀑布/网格卡片列表 | 展示符合筛选条件的 Solution 或 Case 卡片，包含：名称、一句话简介、行业标签、技术标签、交付形式、更新日期 | 点击卡片跳转对应详情页 |
| **空状态提示** | 结果为空反馈 | 当筛选结果为零时展示提示文案与重置建议 | 点击"重置筛选"清除所有过滤条件 |

**URL 参数说明**：
- `?keyword=xxx`：初始化关键词搜索
- `?type=solution`：初始化数据分型为"解决方案"
- `?type=case`：初始化数据分型为"落地案例"
- `?label=xxx`：初始化业务标签过滤

---

### 5.3 案例详情页 `/detail/:id` (Case Detail)

**页面定位**：单个 L2 落地案例（Case）的深度展示页，为售前打单与交付评估提供完整的项目事实。

| 区块名称 | 类型 | 核心内容 |
|----------|------|----------|
| **基本信息头部** | 页面顶部 Banner | 项目名（中/英）、所属行业、保密级别、交付形式（SaaS / 混合 / 私有化）、更新日期 |
| **一句话简介** | 摘要区 | `shortDescription` 字段，一句话概括核心价值 |
| **项目背景** | 富文本正文 | 客户背景、业务痛点、项目规模描述 |
| **技术标签矩阵** | 标签组 | 底层技术（Underlying Tech）+ 业务标签（Labels）的双层标签展示 |
| **交付物说明** | 列表 | 本项目的具体交付形式与产出物说明 |
| **关联方案** | 跳转卡片 | 展示该 Case 归属的 L1 Solution，点击可跳转至 `/solution/:id` |

---

### 5.4 解决方案详情页 `/solution/:id` (Solution Detail)

**页面定位**：单个 L1 核心方案的宏观解析页，面向 BD 与客户汇报场景，偏向"全景架构"展示。

| 区块名称 | 类型 | 核心内容 |
|----------|------|----------|
| **方案头部** | 大 Banner | 方案名称（中/英）、目标行业、核心价值主张 |
| **痛点分析** | 卡片组 | 客户典型痛点的结构化列举 |
| **架构全景** | 图示区 | 技术方案架构图或流程图（可为图片或结构化描述） |
| **能力矩阵** | 标签/卡片 | 该方案包含的全部 L3 原子技术与业务标签 |
| **落地案例列表** | 卡片列表 | 归属本方案的所有 L2 案例，点击跳转 `/detail/:id` |
| **服务对象** | 信息块 | 核心行业与关键决策部门 |

---

### 5.5 技术能力图谱页 `/capabilities` (Capabilities)

**页面定位**：以纯技术视角（剥离客户场景与商业属性）展示底层 AI 原子能力集合，定位为技术交付团队的"能力货架"。

| 区块名称 | 类型 | 核心内容 |
|----------|------|----------|
| **能力全景分组** | 分类卡片网格 | 按技术域分组展示 L3 原子能力标签（如 NLP、CV、语音、工程化等） |
| **能力检索** | 搜索框 | 在原子能力中模糊搜索 |
| **能力详情弹窗/跳转** | 浮层或跳转 | 点击单个能力标签，展示该标签关联的全部 L2 案例列表（跳转至 `/list?label=xxx`） |

## 6. 概念定义与名词解释 (Glossary)

本节对系统中使用的核心术语进行标准化定义，确保产品、研发、运营各方对概念理解一致。

---

### 6.1 数据实体类

| 术语 | 英文 | 定义 |
|------|------|------|
| **核心解决方案** | Solution (L1) | 面向特定大行业或领域的宏观"解题方法论"，包含完整的痛点分析、架构全景和落地路径。是整个系统的顶层内容单元。每个 Solution 可关联多个 Case。 |
| **落地案例** | Case (L2) | L1 方案在具体客户、具体场景中的实施记录。具备明确的交付形式、项目背景、时间节点等可溯源的实施属性。是系统中数量最多的内容单元。 |
| **原子能力标签** | Tech/Business Label (L3) | 支撑 L1 和 L2 的最底层细分技术或业务能力词。以标签形式附着在上层内容实体上，实现跨方案、跨案例的横向聚合与过滤。 |
| **能力货架** | Capability Shelf | 以技术能力为主视角（而非客户场景）组织展示的内容集合，对应 `/capabilities` 页面的定位。 |

---

### 6.2 技术标签类

| 术语 | 英文 | 定义 |
|------|------|------|
| **底层技术** | Underlying Tech | 项目实现所依赖的核心算法或技术框架标签。例：LLM、RAG、知识图谱 (KG)、向量数据库、容器化 (K8s)。不包含业务场景描述。 |
| **业务标签** | Business Label | 描述项目所解决的具体业务场景或能力模块的标签。例：机器翻译、智能分诊、多租户、短视频制作。与底层技术标签互补。 |
| **目标行业** | Target Industry | 案例或方案所服务的客户行业分类。数据来源于 `targetIndustry` 字段，用于列表页的行业维度筛选。 |

---

### 6.3 交付形式类

| 术语 | 英文缩写 | 定义 |
|------|----------|------|
| **SaaS 模式** | SaaS | Software as a Service。以订阅服务形式交付，客户通过云端访问功能，无需独立部署。 |
| **混合模式** | Hybrid | 结合公有云与私有化部署的交付方式，部分功能或数据在客户本地运行，其余通过云端服务提供。 |
| **私有化部署** | On-Premise / Private | 全量系统部署在客户自有基础设施或专属云环境中，数据完全不出客户边界。 |

---

### 6.4 系统功能类

| 术语 | 定义 |
|------|------|
| **多维筛选引擎** | 列表页核心功能模块，支持数据分型（Solution/Case）、行业、业务标签三个维度的组合过滤，以及关键词文本检索与排序规则的叠加应用。 |
| **能力层级结构** | 系统核心数据架构，即 L1 Solution → L2 Case → L3 Label 的三级下钻关系。所有内容的组织、展示与检索逻辑均基于此结构。 |
| **内容分型** | 对系统内数据实体类型的物理区分，即"解决方案（Solution）"与"落地案例（Case）"的区分。影响卡片展示样式与详情页路由。 |
| **动态聚合** | 筛选选项（行业列表、标签列表）不做硬编码，而是在前端/接口层根据当前数据池的实际数据实时聚合生成，确保与数据保持同步。 |
| **保密级别** | Case 实体的属性字段，标记该项目信息对外展示的范围限制。内部展示与外部披露在内容详略上可依据此字段做差异化处理。 |
| **一句话简介** | `shortDescription` 字段的功能性命名，要求用一句话（≤50字）概括项目的核心价值与交付成果，是列表页卡片和搜索匹配的关键字段。 |
| **热门搜索词** | 首页 Hero 区预置的高频检索词快捷入口，由运营人员根据内部访问热度定期维护，当前预置：术语管控、知识图谱、多模态出海、数字孪生。 |
| **Hash Router** | 本项目前端路由实现方式（`HashRouter`），URL 格式为 `域名/#/路径`，适用于不需要服务端路由配置的纯静态部署场景。 |
